## Regression Check: New Issues from Iteration

### New Issue (Minor): `list_issues` not in `allowed-tools` for val, pr, merge

The plan's Phase 2 adds queue-picking logic that requires calling `list_issues` (via the `mcp__plugin_ralph-hero_ralph-github__ralph_hero__list_issues` tool). However, none of the three skills currently include this tool in their `allowed-tools` frontmatter:

- `ralph-val/SKILL.md` allowed-tools: `get_issue`, `create_comment` (no `list_issues`)
- `ralph-pr/SKILL.md` allowed-tools: `get_issue`, `list_sub_issues`, `advance_issue`, `save_issue`, `create_comment` (no `list_issues`)
- `ralph-merge/SKILL.md` allowed-tools: `get_issue`, `list_sub_issues`, `list_dependencies`, `advance_issue`, `save_issue`, `create_comment` (no `list_issues`)

The Phase 2 changes section specifies the queue-picking text to add to each SKILL.md but does not mention adding `list_issues` to the frontmatter `allowed-tools` block. Without this tool permission, the skills will be unable to call `list_issues` at runtime and the queue-picking will fail.

**Severity**: Minor. This is straightforward to catch during implementation -- the implementer will need to add `mcp__plugin_ralph-hero_ralph-github__ralph_hero__list_issues` to the `allowed-tools` in each skill's frontmatter. The fix is mechanical (one line added to each of three files).

**Recommendation**: The implementer should add `list_issues` to the `allowed-tools` for `ralph-val`, `ralph-pr`, and `ralph-merge` as part of Phase 2 implementation.
