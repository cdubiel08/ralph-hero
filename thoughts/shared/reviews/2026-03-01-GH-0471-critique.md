## Minor Observations (Non-blocking)

### 1. Reference count discrepancy: 28 stated, 32 actual

The research says "28 reference occurrences across 7 files." Actual grep count is 32 references across the 7 target skills (ralph-research: 3, ralph-plan: 5, ralph-impl: 6, ralph-review: 8, ralph-split: 4, ralph-triage: 3, ralph-hero: 3). The discrepancy is because some lines were counted differently (e.g., "Per Artifact Comment Protocol in shared/conventions.md" inline text vs link references).

This does not affect the approach — all references in all 7 skills still need to be replaced. The implementation should grep each file first to confirm exact counts before editing.

### 2. quality-standards.md references not covered by plan phases

Research section 4.3 (Discovery 6) explicitly identifies `quality-standards.md` as needing `!cat` treatment — ralph-research, ralph-plan, and ralph-review all use `See [shared/quality-standards.md](../shared/quality-standards.md)` references. These are the same pattern as conventions.md references. The plan's phases do not include this refactor.

The plan's success criteria only checks for `conventions.md` removal and would pass without addressing `quality-standards.md`. This is a valid descoping decision (it's a smaller problem), but the implementer should be aware that `quality-standards.md` will remain as a reference-not-injected file after this issue.

Recommendation: Either add `quality-standards.md` `!cat` refactors to Phase 2 (3 additional skill edits, trivial), or document the explicit deferral in "What We're NOT Doing."

## Conclusion

Plan is comprehensive and well-sourced. The 5-phase sequence (create → refactor → delete → audit → document) is correct — fragments must exist before skill edits, and all references must be gone before deletion. Approve for implementation.
