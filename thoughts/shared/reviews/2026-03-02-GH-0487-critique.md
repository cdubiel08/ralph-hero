# Plan Critique: GH-487 — Enhance computeSuggestedRoster() for Stream-Based Builder Scaling

## Summary

The plan replaces the static `largeSized.length >= 5 ? 2 : 1` builder heuristic in `computeSuggestedRoster()` with a stream-count-driven formula `Math.min(streamCount || 1, 3)`. It threads a new `streamCount` field through `DetectionOptions` -> `detectPipelinePosition` -> `buildResult` -> `computeSuggestedRoster`, and adds a top-level aggregate `suggestedRoster` to the `detect_stream_positions` tool response. Four new tests cover 1, 2, 4 streams and the no-stream fallback.

## Completeness

**Score: Strong**

All layers are covered:
- **Library layer**: `DetectionOptions`, `SuggestedRoster` comment, `computeSuggestedRoster`, `buildResult`, `detectPipelinePosition` (all `buildResult` call sites), `detectStreamPipelinePositions`.
- **Tool layer**: `detect_stream_positions` aggregate roster in response.
- **Test layer**: Four new tests in a dedicated describe block.
- **Success criteria**: Four specific automated checks plus one manual verification.

The plan correctly identifies there are multiple `buildResult` call sites in `detectPipelinePosition` (approximately 14) and instructs the implementer to apply the same pattern to all of them.
