# Plan Critique: GH-477 - Fix MCP_VERSION Mismatch in cli-dispatch.sh

## Verdict: APPROVED

This is a well-scoped, accurately researched plan for a straightforward XS bug fix. No issues found.

## Completeness

The plan defines all necessary changes across exactly two files with three surgical edits:

1. **Immediate fix**: Update `MCP_VERSION` from `"2.4.88"` to `"2.4.95"` in `cli-dispatch.sh` line 5.
2. **Prevention (sed)**: Add a second `sed` command targeting the `MCP_VERSION="X.Y.Z"` format in `release.yml`, placed after the existing sed block (lines 128-130) within the same `mcp_changed` conditional.
3. **Prevention (git add)**: Add `cli-dispatch.sh` to the git staging step in `release.yml` (after line 145), inside the existing `MCP_CHANGED` conditional.

All three changes are necessary and sufficient. The plan correctly identifies that without both prevention steps, the version would drift again on the next release.
