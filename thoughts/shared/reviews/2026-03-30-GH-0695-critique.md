## Findings

### Strengths

**Root cause is correctly identified and fixed at the right layer.** The plan fixes the problem in `ralph-cli.sh` at the `exec just` call site — the only place where the CWD contract is established. Fixing it here means all downstream scripts (`cli-dispatch.sh`, every recipe) are correct without modification.

**Secondary breakage is anticipated.** The plan correctly identifies that `team` and `loop` recipes use bare `./scripts/` paths that would break after the working-directory fix, and provides the exact substitution to `{{justfile_directory()}}/scripts/`. This is the same pattern already used by all other recipes (verified: lines 86, 97, 108, 120, 132, 143, 155, 167, 179, 195, 216 all use `{{justfile_directory()}}/scripts/`).

**Pre-flight probe also fixed.** Adding `--working-directory "$(pwd)"` to the `--summary` probe on line 124 is correct. Although the probe only reads recipe names (not user data), consistency between the probe invocation and the final `exec just` invocation avoids any future divergence issues.

**BATS test skeleton is sound.** The test correctly injects a fake `just` binary at the front of `PATH` via a temp dir, records its argv to stdout, and asserts `--working-directory` appears in the output. The `--summary` guard ensures the pre-flight probe also succeeds with the fake binary. The existing `setup()` pattern (`RALPH_CLI` and `RALPH_JUSTFILE` inherited from `setup()`) is followed.

**Line numbers are verified accurate.** Checked against the live files:
- `ralph-cli.sh:147` — `exec just --justfile "$RALPH_JUSTFILE" "$@"` confirmed
- `ralph-cli.sh:124` — `--summary` probe confirmed
- `justfile:188` — `./scripts/ralph-team-loop.sh` confirmed
- `justfile:209` — `./scripts/ralph-loop.sh` confirmed

### Minor Observation (non-blocking)

The test skeleton captures output from both the pre-flight `--summary` probe invocation and the final `exec just` invocation in a single run. The `echo "ARGS: $*"` line will fire twice (once for `--summary`, once for `status`). The assertion `[[ "$output" =~ "--working-directory" ]]` will still pass (both calls include the flag), so this is harmless. A comment in the test noting "probe fires first, recipe call fires second" would help future readers but is not required.

### Risks

None. The change is a two-word addition to a single `exec` call plus two string substitutions in a justfile. The BATS test provides regression coverage. The existing test suite covers `--version`, `--help`, unknown-command handling, and missing-justfile behaviour — none of which are affected.
