## Shared Constraints

- The fix must not break existing headless or quick-mode behaviour.
- `ralph-cli.sh` is a standalone script installed at `~/.local/bin/ralph`. It has no build step — the file on disk IS the deployed artifact. Tests must cover the file in place via `RALPH_JUSTFILE`.
- `just --justfile` requires `--working-directory` to be passed at the same time; neither flag is optional once the other is present.
- `$(pwd)` in the `exec just` line expands before `exec` replaces the process, so it correctly captures the user's directory at call time.
- The `team` and `loop` recipes use bare `./scripts/` paths that currently resolve against the cache dir (broken after the working-directory fix). They must be updated to use `{{justfile_directory()}}/scripts/`.
- Tests live in `plugin/ralph-hero/scripts/__tests__/ralph-cli.bats` (BATS framework). New tests must follow the existing pattern: set `RALPH_JUSTFILE` in `setup()`, invoke `"$RALPH_CLI"` with `run`.
- No new external dependencies are introduced.

## Current State Analysis

`ralph-cli.sh:147` executes:
```bash
exec just --justfile "$RALPH_JUSTFILE" "$@"
```

`just` with `--justfile` (and no `--working-directory`) silently changes its working directory to the parent directory of the justfile, which is the plugin cache dir (e.g. `~/.claude/plugins/cache/ralph-hero/ralph-hero/2.5.52`). Every recipe inherits this CWD.

Two consequences flow from this:

1. **Interactive mode** — `run_interactive()` in `cli-dispatch.sh:34` runs `exec claude "$cmd"`. Claude Code inherits the wrong CWD, shows a trust prompt for the cache dir, and opens a session there instead of the user's project.

2. **Headless mode** — `run_headless()` in `cli-dispatch.sh:44` runs `git rev-parse --show-toplevel` to capture `repo_root` for the output filter's `vscode://file/` links. With the wrong CWD this either resolves to the cache dir (if it happens to be a git repo) or returns empty.

Additionally, two justfile recipes use relative `./scripts/` paths that would break after the CWD fix:
- `team` recipe (line 188): `./scripts/ralph-team-loop.sh`
- `loop` recipe (line 209): `./scripts/ralph-loop.sh`

All other recipes already use `{{justfile_directory()}}/scripts/` and are unaffected.

## Desired End State

### Verification
- [ ] Running `ralph review -i` from any directory opens Claude Code in that directory, not the plugin cache dir
- [ ] Running `ralph triage` (headless) from a git repo correctly resolves `repo_root` to the user's repo root
- [ ] `ralph team` and `ralph loop` continue to invoke the correct shell scripts by absolute path
- [ ] BATS tests for ralph-cli.bats pass, including a new test verifying `--working-directory` is passed

## What We're NOT Doing

- Not changing `cli-dispatch.sh` logic — the fix is purely at the `just` invocation layer
- Not modifying `run_interactive()` to `cd` explicitly — the correct approach is preserving CWD before `exec just`
- Not changing any skill, recipe, or MCP server code
- Not adding new environment variables or config

## Implementation Approach

There are two files to touch:

1. **`ralph-cli.sh`** — add `--working-directory "$(pwd)"` to the final `exec just` call and to the pre-flight `just --summary` probe.
2. **`justfile`** — update `team` and `loop` recipes to use `{{justfile_directory()}}/scripts/` instead of `./scripts/`.
3. **`ralph-cli.bats`** — add regression test confirming the `--working-directory` flag is passed.

#### Task 1.3: Add BATS regression test for --working-directory flag
- **files**: `plugin/ralph-hero/scripts/__tests__/ralph-cli.bats` (modify)
- **tdd**: true
- **complexity**: low
- **depends_on**: [1.1]
- **acceptance**:
  - [ ] A new test named `"ralph cli passes --working-directory to just"` is added
  - [ ] The test mocks `just` as a shell function that records its arguments, then asserts `--working-directory` appears in the recorded args
  - [ ] The test verifies that the value passed to `--working-directory` is a non-empty string (the user's CWD)
  - [ ] Test follows the existing `setup()` pattern: `RALPH_CLI` and `RALPH_JUSTFILE` are inherited from `setup()`
  - [ ] All existing tests continue to pass

Test skeleton:
```bash
@test "ralph cli passes --working-directory to just" {
    # Create a fake just that records its args and exits cleanly
    local fake_just
    fake_just=$(mktemp)
    chmod +x "$fake_just"
    cat > "$fake_just" <<'SH'
#!/usr/bin/env bash
echo "ARGS: $*"
# Simulate --summary sub-command so pre-flight probe succeeds
if [[ "$*" == *"--summary"* ]]; then echo "status"; fi
exit 0
SH
    PATH_ORIG="$PATH"
    # Inject fake just at front of PATH
    local fake_bin
    fake_bin=$(mktemp -d)
    cp "$fake_just" "$fake_bin/just"
    export PATH="$fake_bin:$PATH"

    run "$RALPH_CLI" status
    export PATH="$PATH_ORIG"
    rm -rf "$fake_bin" "$fake_just"

    [[ "$output" =~ "--working-directory" ]]
}
```
